# LONMA DYNAMIC 全站页眉与页脚统一设计

## 目标

统一现有网站所有正式页面的页眉和页脚，使桌面端、窄分屏和手机端拥有一致的导航体验，同时保留已经确认的首页、案例、案例详情、业务、业务详情、关于和联系页面主体设计。

本轮只处理全站外壳，不重新设计页面内容，不替换当前线上案例总览版本，也不改变业务页现有图片悬浮动画。

## 页面范围

统一覆盖以下页面：

- 首页 `index.html`
- 关于、业务总览、案例总览、联系页面
- Case 01–06 六个案例详情页
- BUILD、PARTS、PHOTO、ECU、CHASSIS、EXHAUST 六个业务详情页

## 页眉设计

### 桌面端

- 保持当前 77px 基准高度和 1900px 内容画布上限。
- 品牌位于左侧，ABOUT、SERVICES、CASES、CONTACT 位于中间，中英切换位于右侧。
- 当前页面导航维持白色文字和蓝色短下划线；悬浮时文字继续保持白色，只出现克制的边缘光。
- 半透明黑灰背景、细灰色底边和现有 BMW 深蓝交互色保持不变。
- 所有页面使用同一组高度、内边距、字体尺寸和交互状态，不让详情页产生另一套页眉比例。

### 窄分屏与手机端

- 小于 768px 时使用两行结构，不使用汉堡菜单。
- 第一行放品牌和中英切换；第二行完整显示四个导航入口。
- 页眉总高度控制在 96–112px 的稳定范围内，避免现有页眉占用过多首屏空间。
- 四个导航入口等宽分布，文字不截断、不换行、不互相遮挡。
- 语言按钮维持现有发光开关样式，但尺寸收紧，不挤压品牌。
- 页面主体从页眉之后自然开始，不使用 `transform: scale()`，不通过负边距补偿。

## 页脚设计

- 复用当前 About、Services、Contact 已使用的三段式 `content-footer` 视觉语言。
- 左侧固定为 `LONMA DYNAMIC`。
- 中间根据语言显示 `龙马态度 · 2026` 或 `AUTOMOTIVE ATTITUDE · 2026`。
- 右侧保留一个与当前页面相关的文字入口：
  - 首页、关于、案例及案例详情：进入联系页面。
  - 业务总览及业务详情：进入联系页面并表达开始项目。
  - 联系页面：返回案例总览。
- 页脚默认使用中性灰边框和白灰文字，仅链接悬浮、键盘焦点和箭头使用 BMW 深蓝。
- 桌面端为三列横向布局；小于 768px 时改为紧凑纵向布局，保持左对齐和清晰点击区域。
- 页脚不使用卡片、粗边框、蓝色模块标题或图片说明条。

## 中英文行为

- 页眉导航、品牌辅助标签、页脚中间文案和页脚入口都跟随现有中英文开关。
- 切换语言后保留在当前页面，不跳回首页。
- 页面刷新后继续沿用现有语言记忆机制。
- 中英文字符串统一使用现有 `data-zh`、`data-en` 或页面现有 i18n 数据方式，不引入第二套状态系统。

## 结构与维护

- 页眉继续使用现有 `.topbar`、`.brand`、`.nav`、`.top-actions` 和 `.lang-toggle` 结构。
- 页脚统一使用 `.content-footer`，并在必要时增加少量语义化修饰类，不复制多套视觉 CSS。
- 公共比例和断点放在现有 `layout-canvas.css`、`styles.css`、`content-pages.css` 中。
- 各页面只补齐必要标记、当前页面状态和翻译属性。
- 不引入框架、组件构建工具或新的运行时依赖。

## 无障碍与交互

- 保留可见的 `:focus-visible` 指示，不用悬浮效果替代键盘焦点。
- 当前导航使用 `aria-current="page"`。
- 导航与页脚链接维持足够的触控区域。
- 中英文切换按钮保留可读的动态 `aria-label`。
- `prefers-reduced-motion` 下关闭非必要发光过渡，但不隐藏状态变化。

## 验收标准

- 所有正式页面都显示一致的页眉和页脚。
- 1900px、1440px、MacBook 分屏窄宽、768px 和手机宽度下没有水平溢出。
- 桌面页眉保持 77px；手机页眉稳定在 96–112px。
- 四个手机导航入口全部可见且可点击。
- 中英文切换在首页、案例总览、案例详情、业务总览、业务详情、关于和联系页面均正常。
- 当前页面导航状态正确，键盘焦点清晰。
- 已确认的主体图片、背景切换、案例交互和业务悬浮图片动画没有视觉或行为回归。
- 自动测试、语法检查和本地页面检查全部通过后才视为完成。

## 非目标

- 不修改案例数据、图片顺序或详细内容。
- 不重新设计业务模块或案例轨道。
- 不增加新的后台、CMS、表单提交服务或 SEO 工作。
- 本轮不自动上传 GitHub 或部署 Vercel；只有用户明确要求上传时再执行。
